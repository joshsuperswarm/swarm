Phase 6 — Harden: generate state server-side, store in cookie, verify in callback

Prompt for Claude Code (paste this)

Goal: Add CSRF protection to OAuth.

Backend
	•	Add GET /auth/github:
	•	Generate random state (UUID).
	•	Set Set-Cookie: gh_oauth_state=<state>; HttpOnly; Secure (if https); SameSite=Lax; Path=/; Max-Age=300.
	•	Return { auth_url: "https://github.com/login/oauth/authorize?client_id=...&redirect_uri=http://localhost:3001/auth/callback&scope=user:email,repo&state=<state>" }.
	•	Update /auth/callback:
	•	Read state from query and cookie.
	•	If missing/mismatch → 400.
	•	Proceed with exchange → upsert user/token → issue app JWT (as in Phase 5).

Frontend
	•	Change LoginPage button to call GET /auth/github and window.location.href = auth_url (instead of the hardcoded link).

How to test
	1.	Normal login → works.
	2.	Manually change state in the URL during redirect → backend should reject with 400.
	3.	Clear cookies and try again → still works.
